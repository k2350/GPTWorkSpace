### 3.9 Framework 1｜数値分解（フェルミ推定）
- **定義**：大きな数値を掛け算の要素に分解して推定する。
- **基本式（需要側）**：市場規模 = 対象人口 × 利用率 × 利用頻度 × 単価
- **基本式（供給側）**：市場規模 = 店舗数（台数）× 稼働率 × 1単位あたり売上
- **使いどころ**：市場規模、売上、利用者数、台数推定
- **典型ミス**：分解が粗い／平均値で一括推定してセグメントを捨てる

### 3.10 Framework 2｜売上・利益分解
- **定義**：業績課題を構造分解してボトルネックを特定する。
- **基本式**：売上 = 客数 × 客単価、利益 = 売上 −（変動費 + 固定費）
- **使いどころ**：売上低下、利益悪化、収益改善
- **典型ミス**：売上だけ議論してコストを見ない／結論先行

### 3.11 Framework 3｜3C分析
- **定義**：Customer・Competitor・Companyの3軸で戦略仮説を整理する。
- **使いどころ**：市場参入、現状分析、競争戦略
- **典型ミス**：3Cを羅列して「だから何か」が出ない

### 3.12 Framework 4｜SWOT → クロスSWOT
- **定義**：内部（S/W）×外部（O/T）から戦略方向を導く。
- **クロスSWOT**：SO（攻勢）/ ST（防衛）/ WO（改善）/ WT（回避）
- **使いどころ**：中長期の戦略オプション比較
- **典型ミス**：分析で終わり、戦略へ接続しない

### 3.13 Framework 5｜PEST分析
- **定義**：政治・経済・社会・技術のマクロ要因を整理する。
- **使いどころ**：新規参入、業界予測、海外展開
- **典型ミス**：列挙のみで自社への影響解釈がない

### 3.14 Framework 6｜4P分析
- **定義**：Product / Price / Place / Promotionで実行設計する。
- **使いどころ**：GTM戦略、販売拡大、施策具体化
- **典型ミス**：顧客ターゲット不在のまま4Pを羅列

### 3.15 Framework 7｜アンゾフマトリクス / バリューチェーン
- **アンゾフ**：既存/新市場 × 既存/新製品で成長方向を整理
- **バリューチェーン**：主活動・支援活動で価値源泉とコストを特定
- **使いどころ**：成長戦略・収益性改善・競争優位分析

### 3.24 3-1. テンプレート設計方針
- **入力を固定化**：問題文・自分の回答・採点基準を毎回同じ順序で渡す。
- **出力形式を固定化**：点数、根拠、改善提案を表形式で返させる。
- **再現性を確保**：採点理由を観点ごとに明示させる。
- **改善につなげる**：次回アクション（何をどう直すか）まで出力させる。
