# ケース面接・フェルミ推定 初級問題集（改訂版アウトライン）

問題集タイトル：
**「ケース面接・フェルミ推定 初級問題集 ― フレームワークで『考える型』を身につける20題 ―」**

---

## 序文

### 本書の目的
ケース面接・フェルミ推定で評価されるのは、答えそのものの正確性よりも、**思考の構造化・論理の一貫性・説明の再現性**です。  
本書は、定番フレームワークを使って20題を解くことで、初学者が「考える型」を実戦レベルで使える状態になることを目指します。

### 対象読者
- 新卒就活生・転職希望者で、ケース面接対策をこれから始める方
- フレームワークの名称は知っているが、実際の使い分けに不安がある方

---

## 第0章：本書の使い方・学習ロードマップ

### 0-1. 各問題の構成

| パート | 内容 |
|---|---|
| ① 問題文 | 面接本番に近い設問 |
| ② ヒント | 使用フレーム、最初の一手、着眼点 |
| ③ 解答例 | 思考プロセス（分解・仮説・検証）と結論 |
| ④ 採点ルーブリック | 4観点×4段階（100点満点）で自己採点 |

### 0-2. 推奨学習ステップ

1. 第1章を通読して、フレームの意味と使いどころを把握する。  
2. 問題文だけを見て最低5分、自力で構造化する。  
3. ヒントで不足観点を確認し、思考の漏れを補う。  
4. 解答例と比較して、分解軸・仮説・結論の差分を明確化する。  
5. ルーブリックで採点し、次に改善すべき観点を1つに絞る。

### 0-3. 3週間ロードマップ（推奨）

- **Week 1（★☆☆）：型を覚える**  
  対象：A-1, A-2, B-1, C-1, D-1  
  目標：1問1フレームで正しく分解する。

- **Week 2（★★☆）：使いこなす**  
  対象：A-3〜5, B-2〜3, C-2〜3, D-2〜3, E-1  
  目標：セグメント分解・複数軸を組み合わせる。

- **Week 3（★★★）：統合する**  
  対象：A-6, B-4, C-4, D-4, E-2  
  目標：複数フレームを統合して一気通貫で提言する。

## 第1章：フレームワーク解説（7フレーム）

> 本章は辞書として利用。問題演習中に必要な項目へ戻って参照する。

### Framework 1｜数値分解（フェルミ推定）
- **定義**：大きな数値を掛け算の要素に分解して推定する。
- **基本式（需要側）**：市場規模 = 対象人口 × 利用率 × 利用頻度 × 単価
- **基本式（供給側）**：市場規模 = 店舗数（台数）× 稼働率 × 1単位あたり売上
- **使いどころ**：市場規模、売上、利用者数、台数推定
- **典型ミス**：分解が粗い／平均値で一括推定してセグメントを捨てる

### Framework 2｜売上・利益分解
- **定義**：業績課題を構造分解してボトルネックを特定する。
- **基本式**：売上 = 客数 × 客単価、利益 = 売上 −（変動費 + 固定費）
- **使いどころ**：売上低下、利益悪化、収益改善
- **典型ミス**：売上だけ議論してコストを見ない／結論先行

### Framework 3｜3C分析
- **定義**：Customer・Competitor・Companyの3軸で戦略仮説を整理する。
- **使いどころ**：市場参入、現状分析、競争戦略
- **典型ミス**：3Cを羅列して「だから何か」が出ない

### Framework 4｜SWOT → クロスSWOT
- **定義**：内部（S/W）×外部（O/T）から戦略方向を導く。
- **クロスSWOT**：SO（攻勢）/ ST（防衛）/ WO（改善）/ WT（回避）
- **使いどころ**：中長期の戦略オプション比較
- **典型ミス**：分析で終わり、戦略へ接続しない

### Framework 5｜PEST分析
- **定義**：政治・経済・社会・技術のマクロ要因を整理する。
- **使いどころ**：新規参入、業界予測、海外展開
- **典型ミス**：列挙のみで自社への影響解釈がない

### Framework 6｜4P分析
- **定義**：Product / Price / Place / Promotionで実行設計する。
- **使いどころ**：GTM戦略、販売拡大、施策具体化
- **典型ミス**：顧客ターゲット不在のまま4Pを羅列

### Framework 7｜アンゾフマトリクス / バリューチェーン
- **アンゾフ**：既存/新市場 × 既存/新製品で成長方向を整理
- **バリューチェーン**：主活動・支援活動で価値源泉とコストを特定
- **使いどころ**：成長戦略・収益性改善・競争優位分析

## 第2章付属：共通採点ルーブリック（全20題共通）

### 配点設計（100点満点）

| 観点 | 評価の問い | 配点 |
|---|---|---|
| ① 問題の構造化 | 論点をMECEに分解できているか | 25点 |
| ② フレーム活用 | 問題に適切なフレームを選び使えているか | 25点 |
| ③ 論理の一貫性 | 分析〜結論が一本で繋がっているか | 25点 |
| ④ 数値感覚・具体性 | 根拠ある数値と具体的提言があるか | 25点 |

### 総合評価の目安

| 合計点 | 評価 | 解釈 |
|---|---|---|
| 90〜100 | ★★★ | 本番対応可能 |
| 75〜89 | ★★☆ | 合格圏、深さを伸ばす段階 |
| 50〜74 | ★☆☆ | 反復練習で型を定着 |
| 0〜49 | 要基礎固め | 第1章へ戻って再演習 |

### カテゴリ別の重点観点

| カテゴリ | 重点 |
|---|---|
| A. フェルミ推定 | ①構造化 + ④数値感覚 |
| B. 売上・利益分析 | ①構造化 + ③論理一貫性 |
| C. 環境分析 | ②フレーム活用 + ③論理一貫性 |

## 第3章（追加）：生成AIで採点ルーブリックに沿って評価させるプロンプトテンプレート

> 本章は、学習者が自分の回答を生成AIに採点させる際、評価のばらつきを減らすための実務テンプレート集です。

### 3-1. テンプレート設計方針

- **入力を固定化**：問題文・自分の回答・採点基準を毎回同じ順序で渡す。  
- **出力形式を固定化**：点数、根拠、改善提案を表形式で返させる。  
- **再現性を確保**：採点理由を観点ごとに明示させる。  
- **改善につなげる**：次回アクション（何をどう直すか）まで出力させる。

### 3-2. 汎用プロンプトテンプレート（コピペ用）

```text
あなたはケース面接トレーナーです。
以下の「問題」「受験者回答」「採点ルーブリック」に基づいて、厳密に採点してください。

# 採点ルール
- 観点は4つ（各25点、合計100点）
  1) 問題の構造化
  2) フレームワークの適切な活用
  3) 仮説・論理の一貫性
  4) 現実的な数値感覚・具体性
- 各観点について、点数・評価理由・改善提案を必ず出す
- 甘くしすぎず、根拠ベースで評価する
- 不明点は推測せず「不足情報」と明記する

# 出力形式（厳守）
1. 総合点（100点満点）
2. 観点別採点表（観点 / 点数 / 主な根拠 / 改善アクション）
3. 良かった点（3つ）
4. 改善優先度の高い課題（3つ）
5. 次回回答のための30秒アドバイス

# 問題
{{QUESTION}}

# 受験者回答
{{ANSWER}}

# 採点ルーブリック
{{RUBRIC}}
```

### 3-3. カテゴリ別カスタムテンプレート

- **A（フェルミ推定）向け追記**  
  「分解式の妥当性」「オーダー感」「クロスチェック有無」を必須評価項目にする。

- **B（売上・利益分析）向け追記**  
  「売上・コスト両面の分解」「ボトルネック特定」「打ち手の因果」を必須評価項目にする。

- **D（戦略立案）向け追記**  
  「選択しなかった戦略案との比較」「実行優先順位」「KPI定義」を必須評価項目にする。

### 3-4. 低品質な採点を避けるための禁止事項（プロンプト内に追記）

```text
- 抽象論だけで採点しない
- 点数と評価コメントに矛盾を作らない
- 改善提案は「誰が・何を・いつまでに」の形式で書く
- 回答にない内容を勝手に補完して高得点化しない
```

### 3-5. 採点結果の活用テンプレート（学習ログ用）

```text
【今回の総合点】
【最も低かった観点】
【原因（自分の振り返り）】
【次回の改善アクション（1つだけ）】
【次に解く問題番号】
```
